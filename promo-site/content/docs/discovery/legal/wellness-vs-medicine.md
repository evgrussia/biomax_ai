# Разграничение Wellness и Медицины для BIOMAX AI

## Executive Summary

BIOMAX AI позиционируется как **wellness-платформа** (оздоровительный сервис), а не как медицинское учреждение. Это принципиальное разграничение определяет правовой статус, допустимые функции и юридическую ответственность. Ключевой принцип: **информирование, а не диагностика; рекомендации, а не назначения**.

---

## 1. Правовая база

### 1.1 Медицинская деятельность (определение)

**Федеральный закон № 323-ФЗ "Об основах охраны здоровья граждан":**

> **Медицинская деятельность** — профессиональная деятельность по оказанию медицинской помощи, проведению медицинских экспертиз, медицинских осмотров и медицинских освидетельствований, санитарно-противоэпидемических (профилактических) мероприятий и профессиональная деятельность, связанная с трансплантацией органов и тканей, обращением донорской крови и её компонентов.

**Медицинская помощь включает:**
- Диагностику заболеваний
- Лечение заболеваний
- Профилактику заболеваний (в медицинском смысле)
- Реабилитацию
- Медицинские осмотры

### 1.2 Требования к медицинской деятельности

| Требование | Основание |
|------------|-----------|
| **Лицензия** | ФЗ № 99-ФЗ "О лицензировании" |
| **Медицинское образование** | ФЗ № 323-ФЗ, Приказ Минздрава |
| **Аккредитация специалиста** | ФЗ № 323-ФЗ |
| **Соответствие стандартам** | Приказы Минздрава |
| **Регистрация в Росздравнадзоре** | Для медизделий с ИИ |

### 1.3 Оздоровительная деятельность (Wellness)

**Оздоровительная деятельность** — деятельность, направленная на поддержание и укрепление здоровья, **не связанная с диагностикой и лечением заболеваний**.

**Примеры wellness-услуг (без лицензии):**
- Консультации по питанию (нутрициология)
- Фитнес и персональные тренировки
- Спа-процедуры (косметические)
- Коучинг здорового образа жизни
- Мониторинг показателей (без диагностики)
- Рекомендации по добавкам (БАД)

## 2. Ключевые различия

### 2.1 Wellness vs Medicine: сравнительная таблица

| Аспект | Медицинская деятельность | Wellness / BIOMAX AI |
|--------|--------------------------|----------------------|
| **Цель** | Диагностика и лечение заболеваний | Оптимизация здоровья, профилактика |
| **Субъект** | Пациент (больной) | Клиент (здоровый человек) |
| **Результат** | Диагноз, назначение лечения | Информация, рекомендации |
| **Лицензия** | Обязательна | Не требуется |
| **Ответственность** | Медицинская + уголовная | Гражданско-правовая |
| **Стандарты** | Клинические рекомендации, протоколы | Нет обязательных стандартов |
| **Документация** | Медицинская карта | Личный кабинет пользователя |

### 2.2 Примеры разграничения

| Запрос пользователя | Медицинский ответ ❌ | Wellness ответ ✅ |

| "Какие добавки пить для сна?" | "Вам нужен золпидем" | "Для улучшения сна могут быть полезны магний, глицин, мелатонин. Проконсультируйтесь с врачом перед приёмом" |

### 3.1 ЗЕЛЁНАЯ ЗОНА: Допустимо без ограничений

| Категория | Примеры | Обоснование |
|-----------|---------|-------------|

| **Tracking привычек** | "Вы достигли цели по шагам" | Мониторинг активности |
| **Напоминания** | "Время принять добавки" | Сервисная функция |
| **Нутрициология** | "Продукты, богатые магнием: орехи, зелень" | Информация о питании |

### 3.2 ЖЁЛТАЯ ЗОНА: Допустимо с disclaimers

| Категория | Примеры | Обязательные disclaimers |
|-----------|---------|--------------------------|
| **Интерпретация анализов** | "Ваш витамин D ниже оптимального диапазона" | "Обратитесь к врачу для точной интерпретации" |
| **Рекомендации по добавкам** | "Рассмотрите приём omega-3 2г/день" | "Проконсультируйтесь с врачом, это не медицинский совет" |
| **Рекомендации по питанию** | "Для вашего профиля может подойти кето-диета" | "Перед изменением диеты проконсультируйтесь со специалистом" |
| **Рекомендации по тренировкам** | "Ваш HRV указывает на недовосстановление" | "При наличии заболеваний проконсультируйтесь с врачом" |
| **Longevity протоколы** | "Протокол NMN может поддержать NAD+" | "Это экспериментальный протокол, не замена лечению" |
| **Интерпретация генетики** | "У вас есть вариант MTHFR C677T" | "Генетическая информация требует интерпретации специалиста" |

### 3.3 КРАСНАЯ ЗОНА: Категорически запрещено

| Категория | Примеры | Почему запрещено |
|-----------|---------|------------------|
| **Постановка диагнозов** | "У вас диабет 2 типа" | Медицинская деятельность без лицензии |
| **Назначение рецептурных препаратов** | "Принимайте метформин 500мг 2р/день" | Назначение лекарств = медицинская помощь |
| **Отмена назначений врача** | "Вам не нужен этот препарат" | Вмешательство в лечение |
| **Инъекционные протоколы** | "Колите гормон роста 2IU в день" | Опасно без медицинского контроля |
| **Диагностика по симптомам** | "Ваши симптомы указывают на рак" | Диагностика = медицинская деятельность |
| **Советы для беременных** | "Принимайте эти добавки" | Высокий риск, требуется врач |
| **Советы для детей** | Любые персонализированные советы | Особая уязвимая категория |
| **Психиатрические состояния** | "У вас депрессия, попробуйте..." | Требуется квалифицированная помощь |

## 4. Ответственность за незаконную медицинскую деятельность

### 4.1 Статья 235 УК РФ

> **Незаконное осуществление медицинской деятельности** — осуществление медицинской деятельности лицом, не имеющим лицензии на данный вид деятельности, при условии, что такая лицензия обязательна.

**Санкции:**

| Последствия | Наказание |
|-------------|-----------|
| Причинение вреда здоровью | До 3 лет лишения свободы |
| Смерть по неосторожности | До 5 лет лишения свободы |

### 4.2 Административная ответственность

| Нарушение | Штраф | Статья КоАП |
|-----------|-------|-------------|
| Оказание услуг без лицензии | 40 000 - 50 000 ₽ (ИП), 170 000 - 250 000 ₽ (юрлица) | ст. 14.1 ч. 2 |
| С причинением вреда | До 350 000 ₽ + конфискация | ст. 14.1 ч. 3 |

### 4.3 Судебная практика (примеры)

**Кейс 1:** Нутрициолог назначала "лечебные протоколы" и "диагностировала" заболевания по анализам → административное дело, штраф.

**Кейс 2:** Онлайн-сервис давал "диагнозы" по симптомам без врачебного участия → блокировка сайта Роспотребнадзором.

**Кейс 3:** БАД рекламировался как "лечение диабета" → штраф ФАС + отзыв регистрации.

**Вывод для BIOMAX AI:** Чёткое позиционирование как wellness + disclaimers критически важны.

## 5. Сравнение с международными практиками

### 5.1 FDA (США) — Digital Health Guidelines

FDA разделяет цифровые продукты для здоровья на категории:

| Категория | Регулирование | Примеры |
|-----------|---------------|---------|
| **Low-risk wellness** | Не регулируется FDA | Фитнес-трекеры, apps для ЗОЖ |
| **Software as Medical Device (SaMD)** | Требует регистрации | AI для диагностики |
| **Clinical Decision Support** | Частично регулируется | Системы поддержки решений врача |

**FDA Wellness Policy (2019):**
- Apps для "general wellness" не требуют регистрации, если:
  - Не предназначены для диагностики/лечения
  - Низкий риск для пользователя
  - Направлены на поддержание здоровья

**Применение к BIOMAX AI:** Аналогичное позиционирование — "general wellness" без диагностических функций.

### 5.2 EU MDR (Европа)

| Класс | Риск | Примеры | Регулирование |
|-------|------|---------|---------------|
| I | Низкий | Фитнес-apps | Минимальное |
| IIa | Низкий-средний | Apps для мониторинга | Сертификация |
| IIb | Средний-высокий | Apps для диагностики | Строгая сертификация |
| III | Высокий | Имплантируемые устройства | Максимальный контроль |

**BIOMAX AI = Класс I (при соблюдении ограничений)**

### 5.3 Ключевые принципы для compliance

Из международной практики выводим принципы:

1. **Не претендовать на диагностику** — только информирование
2. **Не заменять врача** — направлять к специалистам
3. **Disclaimers везде** — юридическая защита
4. **Human-in-the-loop** — для критичных решений
5. **Transparency** — объяснять ограничения AI

## 8. Процедура review рекомендаций

### 8.1 Категоризация рисков

| Риск | Категория рекомендаций | Review |
|------|------------------------|--------|
| **Низкий** | Общая информация, трекинг | Автоматически |
| **Средний** | Добавки, диеты, упражнения | AI Safety check |
| **Высокий** | Интерпретация анализов, longevity | AI Safety + human review (опционально) |
| **Критический** | Red flags, экстренные состояния | Автоматическое направление к врачу |

### 8.2 Процесс human-in-the-loop (для Enterprise)

```

│                                                                  │

│                    │                                             │
```
