# Анализ требований 152-ФЗ для BIOMAX AI

## Executive Summary

Платформа BIOMAX AI обрабатывает **специальные категории персональных данных** (здоровье, генетика, биометрия), что требует строгого соблюдения 152-ФЗ "О персональных данных" и смежного законодательства. Ключевые риски связаны с локализацией данных, получением согласий и интеграцией с зарубежными сервисами.

---

## 1. Классификация данных BIOMAX AI по 152-ФЗ

### 1.1 Персональные данные (ст. 3 152-ФЗ)

| Категория данных | Примеры в BIOMAX AI | Классификация |
|-----------------|---------------------|---------------|
| **Идентификационные** | ФИО, email, телефон, Telegram ID | Общие ПД |
| **Данные о здоровье** | HRV, пульс, сон, глюкоза, результаты анализов | **Специальные ПД** (ст. 10) |
| **Генетические данные** | SNP-профили, результаты 23andMe, Nebula | **Специальные ПД** (ст. 10) |
| **Биометрические данные** | Голос (если используется), фото (если анализируется) | **Биометрические ПД** (ст. 11) |
| **Медицинская история** | Загруженные анализы, протоколы лечения | **Специальные ПД** |
| **Поведенческие данные** | Активность, питание, привычки | Общие ПД |

### 1.2 Специальные категории ПД (ст. 10 152-ФЗ)

> **Обработка данных о состоянии здоровья по общему правилу ЗАПРЕЩЕНА**, за исключением случаев, прямо предусмотренных законом.

**Исключения, применимые к BIOMAX AI:**
1. ✅ **Письменное согласие субъекта** — основной правовой базис
2. ✅ **Защита жизни/здоровья** — для экстренных ситуаций (red flags)
3. ❌ Медицинские цели — НЕ применимо (нет медицинской лицензии)

## 2. Требования к согласию на обработку ПД

### 2.1 Форма согласия (ст. 9 152-ФЗ)

Для **специальных категорий ПД** требуется **письменное согласие** в форме:
- Электронного документа, подписанного ЭЦП, или
- Простой электронной подписи (галочка + подтверждение по SMS/email)

### 2.2 Обязательные элементы согласия

```
1. ФИО субъекта персональных данных
2. Адрес субъекта / данные документа, удостоверяющего личность
3. Наименование и адрес оператора
4. Цель обработки персональных данных
5. Перечень персональных данных, на обработку которых даётся согласие
6. Перечень действий с персональными данными (сбор, хранение, передача и т.д.)
7. Срок, в течение которого действует согласие
8. Способ отзыва согласия
9. Подпись субъекта
```

### 2.3 Рекомендуемая структура согласия для BIOMAX AI

**Минимум 3 отдельных согласия:**

| Согласие | Данные | Обязательность |
|----------|--------|----------------|
| **Согласие 1: Базовое** | Идентификационные данные, поведенческие | Обязательно для регистрации |
| **Согласие 2: Данные о здоровье** | Wearables, анализы, медицинская история | Опционально (для расширенных функций) |
| **Согласие 3: Генетические данные** | ДНК-профили, SNP, микробиом | Опционально (отдельное согласие) |

## 3. Локализация данных (ст. 18 152-ФЗ)

### 3.1 Текущие требования (с 01.07.2025)

> **КРИТИЧЕСКИ ВАЖНО**: С 1 июля 2025 года ужесточены требования по локализации.

**Новые требования (ФЗ от 28.02.2025 № 23-ФЗ):**

| Операция | Требование до 01.07.2025 | Требование с 01.07.2025 |
|----------|--------------------------|-------------------------|
| Первичный сбор | В России | В России |
| Хранение | Можно копия за рубежом | **Только в России** |
| Обработка | Можно за рубежом | **Только в России** |
| Трансграничная передача | С уведомлением РКН | С уведомлением РКН (ограничено) |

### 3.2 Применение к BIOMAX AI

**ПРОБЛЕМНЫЕ ОБЛАСТИ:**

| Интеграция | Проблема | Решение |
|------------|----------|---------|
| **23andMe, Nebula** | Данные хранятся в США | Пользователь сам загружает raw data; BIOMAX хранит только результаты анализа |
| **Oura, WHOOP** | Серверы за рубежом | Синхронизация через API → хранение в РФ, не хранение на зарубежных серверах |
| **OpenAI GPT-4** | Данные передаются в США | Анонимизация данных перед передачей ИЛИ использование российских LLM (GigaChat) |
| **Dexcom, Libre** | CGM данные | Аналогично wearables — импорт и хранение в РФ |

## 4. Трансграничная передача данных (ст. 12 152-ФЗ)

### 4.1 Порядок уведомления Роскомнадзора

**До начала трансграничной передачи** необходимо подать уведомление, содержащее:

1. Наименование и адрес оператора
2. Номер ранее поданного уведомления об обработке ПД
3. Правовое основание трансграничной передачи
4. Цели передачи данных
5. Категории и перечень передаваемых ПД
6. Категории субъектов ПД
7. Перечень иностранных государств-получателей
8. Дата оценки соблюдения конфиденциальности получателем

### 4.2 Страны-получатели по категориям

| Категория | Страны | Требования |
|-----------|--------|------------|
| **Адекватная защита** | Страны ЕС, Швейцария, Канада, Израиль и др. | Упрощённый порядок |
| **Без адекватной защиты** | США (большинство сервисов) | Дополнительные меры защиты + уведомление |

**Для BIOMAX AI**: Большинство интеграций (OpenAI, 23andMe, Oura) — США → требуется полная процедура уведомления.

✅ **BIOMAX AI ОБЯЗАН** подать уведомление, так как:
- Обрабатывает специальные категории ПД

## 11. Рекомендации по архитектуре для compliance

### 11.1 Принципы проектирования

1. **Privacy by Design** — защита ПД на этапе проектирования
2. **Data Minimization** — собирать только необходимые данные
3. **Purpose Limitation** — использовать данные только для заявленных целей
4. **Storage Limitation** — удалять данные по истечении срока

### 11.2 Рекомендуемая архитектура хранения

```yaml
data_storage:
  location: "Russia (Yandex Cloud / VK Cloud / on-premise)"
  
  databases:
    profiles:
      type: "PostgreSQL"
      encryption: "AES-256 at rest"
      contains: "user profiles, consents, health data"
      
    time_series:
      type: "ClickHouse"
      encryption: "AES-256 at rest"
      contains: "wearables data, CGM, activity"
      
    vectors:
      type: "Qdrant"
      encryption: "AES-256 at rest"
      contains: "embeddings (anonymized)"
      
  external_integrations:
    llm_requests:
      provider: "GigaChat (primary) / OpenAI (anonymized only)"
      data_sent: "anonymized queries without PII"
      
    wearables_sync:
      method: "user-initiated import"
      storage: "only in Russian infrastructure"
```

### 11.3 Анонимизация для LLM-запросов

```python
# Пример анонимизации перед отправкой в OpenAI
def anonymize_for_llm(user_data: dict, query: str) -> str:
    """
    Анонимизирует данные пользователя перед отправкой в LLM.
    Заменяет ПД на плейсхолдеры.
    """
    anonymized = {
        "age_group": get_age_group(user_data["birth_date"]),  # "30-35"
        "sex": user_data["sex"],  # "M" / "F"
        "metrics": {
            "hrv": user_data["hrv"],  # числовые значения без привязки
            "sleep_score": user_data["sleep_score"],
            "glucose_avg": user_data["glucose_avg"],
        }
    }
    # Исключаем: имя, email, телефон, точный возраст, адрес
    return f"User profile: {anonymized}\n\nQuery: {query}"
```
